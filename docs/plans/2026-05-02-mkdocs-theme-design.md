# MkDocs Theme Refresh Design

**Date:** 2026-05-02

## Goal

在现有教程站点结构不变的前提下，提升 MkDocs 站点的视觉质量，重点解决暗色模式不适配、移动端阅读体验一般、中文教程正文排版偏生硬的问题。

## Scope

本次只调整站点展示层：

- 切换主题实现
- 增加明暗配色支持
- 增加适合教程阅读的中文排版样式
- 优化移动端导航与正文表现

不修改文档正文内容，不调整章节导航结构，不新增复杂交互，也不继续处理 `NOTES.md` 的翻译问题。

## Constraints

- 当前站点已经用 `mkdocs.yml` 和 `docs/` 软链接镜像成功构建，不能破坏现有文档路径。
- 站点主要承载长篇教程阅读，样式要服从可读性，而不是展示型首页。
- `64_6809_Target/docs/NOTES.md` 仍需在附录中忠实显示，但不需要单独视觉包装。
- 已安装 `material`，因此优先利用现有能力，不引入额外运行时复杂度。

## Chosen Approach

采用 `mkdocs-material` 作为基础主题，在其现成的暗色模式、移动端抽屉导航、目录联动和代码块能力之上，只补一层小规模自定义 CSS。

这样做的原因是：

- 暗色模式和移动端体验由成熟主题兜底，避免手工覆盖默认 `mkdocs` 的大量细节。
- 站点配置仍然保持简单，后续维护成本低。
- 中文正文的排版优化可以用少量局部 CSS 完成，不需要重写模板。

## Rejected Alternatives

### 1. 继续使用默认 `mkdocs` 主题并手工补样式

不采用。虽然主题切换改动最小，但暗色、移动端导航、代码块、目录联动都需要额外修补，投入与结果不成比例。

### 2. 直接做更重的品牌化定制

本轮不采用。首页模板、复杂卡片布局、额外脚本和品牌风格可以留到后续第二轮，如果 `Material` 的基础能力仍然不够，再升级到那一步。

## Visual Direction

视觉方向定为“克制、清晰、面向长文阅读”：

- 默认浅色主题保持高对比度，但避免过亮的纯白感
- 深色主题提升正文与代码块的层次，避免整页发灰
- 标题、正文、列表、代码块之间的间距更稳定
- 让移动端优先保证导航可达和段落舒展，而不是压缩信息密度

## Theme Features

计划启用的主题能力：

- 明暗模式切换
- 顶部导航栏与移动端抽屉导航
- 当前章节高亮
- 页面内目录跟随
- 返回顶部
- 代码块复制按钮

这些能力都属于教程站点的低风险增强，不改变文档本身。

## Styling Plan

新增 `docs/stylesheets/extra.css`，只做以下范围的样式覆盖：

- 控制正文最大宽度、行高和段落间距
- 调整标题层级、列表缩进和引用块边距
- 优化代码块、表格、图片在小屏幕上的可读性
- 微调导航与内容区间距，减少默认风格的生硬感

不引入自定义 JavaScript，不改 HTML 模板。

## Mobile Behavior

移动端主要依赖 `Material` 的原生导航抽屉与粘性头部。自定义样式只负责：

- 缩小过宽的页面边距
- 提高代码块和表格横向滚动体验
- 压缩过大的标题和空白
- 保证附录和长文页面不会在小屏上显得拥挤

## Verification

最低验收标准：

- `mkdocs build --strict` 成功
- 明暗模式都能正常构建且样式没有明显冲突
- 抽查首页、普通章节、`64_6809_Target/Readme.md` 和 `NOTES.md`
- 抽查移动端宽度下代码块、表格、导航和正文间距是否可读
