# MkDocs Theme C Refresh Design

**Date:** 2026-05-02

## Goal

在现有教程站点 A 版主题的基础上，升级为更完整的 C 版展示层：保留现有文档结构与导航语义，但让首页更像真正的教程入口、章节导航更容易浏览、代码块具备语法高亮，并让深浅模式默认跟随系统主题。

## Why A Wasn't Enough

A 版已经解决了基础的暗色主题和移动端排版问题，但仍有几个明显缺口：

- 代码块没有按 `mkdocs-material` 的标准能力启用语法高亮
- 深浅模式只能手动切换，不能自动跟随系统偏好
- 首页仍然过于接近“普通文档页”，缺少教程入口感
- 章节访问依然过度依赖左侧导航树，第一次进入时不够直观

因此，这一轮不只是“继续调 CSS”，而是把站点做成更明确的教程型入口页。

## Scope

本轮只改站点展示层和 Markdown 渲染配置：

- 启用 `mkdocs-material` 官方推荐的自动深浅模式切换
- 启用 fenced code block 的语法高亮扩展
- 增加模板覆盖，使首页出现教程导览区、章节入口区和附录入口区
- 强化导航样式与章节目录的浏览体验
- 继续优化移动端首页和正文布局

不修改教程正文内容，不改变现有章节路径，不新增搜索插件或运行时脚本框架。

## Constraints

- 必须保留当前 `docs/` 软链接镜像方案，避免复制正文内容
- 顶层 `Readme.md` 仍然是首页内容来源之一，不能把它彻底替换成另一份独立文档
- `NOTES.md` 继续保留在附录位置，不做翻译，但要有更明确的入口
- 模板定制要尽量浅层，避免完全脱离 `Material` 的可升级路径

## Chosen Approach

采用“官方能力优先、轻模板增强、首页导览加强”的方案：

1. 用 `theme.palette` 的 `media` 配置启用 `system -> light -> dark` 三段式切换
2. 用 `pymdownx.highlight`、`pymdownx.inlinehilite`、`pymdownx.superfences` 等扩展启用代码高亮
3. 用 `theme.custom_dir: overrides` 添加轻量模板覆盖
4. 在首页模板中注入：
   - 教程简介英雄区
   - 分阶段章节入口
   - 附录和补充资源入口
5. 用 CSS 统一这些模块的视觉风格，并继续照顾移动端

这样做的优点是：

- 自动深浅模式和代码高亮都遵循官方文档，不靠脆弱 hack
- 首页明显更像“教程站点”，而不是“一个很长的 README”
- 章节入口不再只能靠左侧树导航，首次浏览路径更清楚

## Information Architecture

站点整体结构不变，但首页承担更多“导览”职责：

- 顶部：教程定位与阅读方式说明
- 中部：按阶段划分的章节入口
  - 起步：00-13
  - 语义与数据：14-33
  - 完善与扩展：34-64
- 底部：附录入口与项目状态说明

左侧导航仍然保留完整章节树，适合线性阅读；首页则变成“第一次进入时的总入口”。

## Navigation Behavior

- 桌面端继续保留左侧导航与右侧目录
- 首页增加明显的快捷卡片入口，降低对侧边栏的依赖
- 移动端优先让首页入口模块更紧凑，避免一上来就是密集树形导航
- 附录入口在首页也出现一次，避免用户不知道 `NOTES.md` 的存在

## Visual Direction

在 A 版“克制阅读感”的基础上，这一轮更强调“教程门户”：

- 首页头部更有层次，不再只是普通一级标题
- 章节入口使用卡片或网格布局，提高扫描效率
- 正文页继续保持安静、稳定、偏阅读的样式
- 深色模式要更像原生支持，而不是浅色模式的反相近似

## Implementation Surface

预计涉及：

- `mkdocs.yml`
- `docs/stylesheets/extra.css`
- `overrides/` 下的模板文件
- 如有必要，少量首页辅助样式或图标配置

## Verification

最低验收标准：

- `mkdocs build --strict` 成功
- 代码块 HTML 带有高亮相关类名，视觉上具备语法高亮
- 在支持 `prefers-color-scheme` 的环境下，首次进入默认跟随系统主题
- 首页包含新的导览区与章节入口区
- 手机宽度下首页卡片、导航抽屉、代码块滚动与章节页排版都可用
