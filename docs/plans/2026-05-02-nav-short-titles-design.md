# Navigation Short Titles Design

## Context

当前左侧导航的“教程”部分仅显示 `第 00 部分`、`第 01 部分` 这类编号标签。虽然顺序清楚，但无法直接表达章节主题，用户需要点击进去或查看正文标题才能知道具体内容。在移动端和窄侧栏里，这种纯编号导航的信息密度偏低。

## Goal

将教程导航从“仅编号”改成“仅短标题”，让用户在左侧即可快速识别每一部分的主题，同时避免过长标题挤占移动端空间。

## Approaches

### A. 直接维护 `mkdocs.yml` 里的短标题

为 00 到 64 每一项手工写一个短标题，替换现有的 `第 xx 部分` 标签。

优点：
- 最稳定，完全走 MkDocs 原生导航
- 不依赖运行时脚本，对 Via 等兼容性较弱浏览器也友好
- 站点不会再频繁更新，一次维护成本可接受

缺点：
- 需要手工维护一份标题映射
- 后续如果正文标题变化，导航不会自动同步

### B. 用前端脚本动态替换导航文案

保留 `mkdocs.yml` 编号标签，在页面加载后按路径或标题映射替换左侧导航文本。

优点：
- 配置文件改动较少

缺点：
- 引入额外前端逻辑
- 兼容性和可维护性更差
- 对当前问题属于过度设计

### C. 从文档一级标题自动生成导航

写额外生成脚本或插件，从每篇文档的 `# 标题` 自动抽取并压缩成短标题。

优点：
- 理论上更自动化

缺点：
- 超出 MkDocs 默认能力
- 复杂度明显高于这次需求
- 对一个基本不会再更新的教程没有必要

## Recommendation

采用 A。教程内容基本稳定，手工维护导航短标题是最稳妥、最清晰的方案。它没有运行时兼容性风险，也不需要额外构建逻辑。

## Scope

- 只修改 `mkdocs.yml` 中“教程”分组的 00 到 64 标签
- 首页、附录、文档正文标题不变
- 导航标签不再显示“第几部分”，改为短主题名称
- 过长主题人工压缩为适合侧栏阅读的短标题

## Verification

- `mkdocs build --strict`
- 新增导航标签检查脚本，验证生成站点不再使用 `第 xx 部分` 这类旧标签
- `bash tools/check_mobile_nav.sh`
- `bash tools/check_homepage_links.sh`
- `bash tools/check_browser_compat_notice.sh`
